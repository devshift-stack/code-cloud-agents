# Vollständige Feature-Entwicklung

## Checkliste für JEDES Feature
- [ ] **Backend** - API, Logik, Datenbank
- [ ] **Frontend** - React Components, Pages
- [ ] **UI/UX Design** - Layout, Responsiveness, User Flow
- [ ] **Styling** - CSS, Tailwind, Themes
- [ ] **Types** - TypeScript Interfaces
- [ ] **Tests** - Unit, Integration
- [ ] **Error Handling** - Edge Cases, User Feedback

## UI/UX Priorität
- Frontend und Design sind GLEICHWERTIG zu Backend
- Niemals "Backend fertig, Frontend kommt später"
- User Experience zuerst denken
- Mobile/Responsive nicht vergessen

## Frontend ↔ Backend Verbindung

### 1. API Design
- Endpoints definieren (GET, POST, PUT, DELETE)
- Request/Response Types erstellen
- Error Responses planen

### 2. Backend Implementation
- Route Handler erstellen
- Business Logic implementieren
- Datenbank-Schema wenn nötig

### 3. Frontend Integration
- API Client/Fetch Functions
- Loading States
- Error Handling im UI
- Success Feedback

### 4. Testing
- API manuell testen (curl/Postman)
- Frontend mit echtem Backend testen
- Edge Cases prüfen

## Verboten
- UI/UX ignorieren oder aufschieben
- Frontend ohne Backend-Anbindung als "fertig" bezeichnen
