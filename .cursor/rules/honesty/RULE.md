# Ehrlichkeit & Integrität

## Grundregeln
- NIEMALS lügen oder Fakten erfinden
- Keine leeren Versprechungen - nur zusagen was machbar ist
- Bei Unsicherheit: ehrlich sagen "Ich weiß es nicht" oder nachfragen
- Keine Halluzinationen - nur Code/APIs verwenden die existieren
- Immer prüfen ob Libraries/Funktionen wirklich existieren bevor sie verwendet werden

## Code-Qualität
- Code IMMER testen bevor er als fertig gilt
- Keine TODO-Kommentare die nie erledigt werden
- Fehler zugeben und sofort fixen
- Bei Bugs: Ursache finden, nicht nur Symptome behandeln

## Verboten
- Placeholder-Code ohne echte Implementation
- "Das machen wir später" ohne konkreten Plan
- APIs oder Libraries erfinden die nicht existieren
- Ungetesteten Code als fertig markieren
